# 🚀 Pulse Project Deployment

Данный репозиторий содержит конфигурацию для развертывания проекта **Pulse** с помощью Docker Compose. Архитектура включает в себя бэкенд на Python, фронтенд на Node.js/NPM, брокер очередей Redis, а также воркер и планировщик Celery.

---

## 🛠 1. Настройка окружения (.env)

Перед запуском проекта обязательно создайте файл `.env` в корневой директории проекта и заполните его следующими параметрами:

```env
# --- Настройки сети Docker ---
NETWORK_NAME=testpulse-net

# --- Порты для внешнего доступа ---
BACK_PORT=7000
FRONT_PORT=3000

# --- Имена контейнеров ---
BACK_NAME=testpulseback_container
FRONT_NAME=testfrontpulse_container
REDIS_NAME=testpulse_redis
CELERY_CURRENCY_NAME=testpulse_celery_worker
CELERY_BEAT_NAME=testpulse_celery_beat
```

> ⚠️ **Важно:** Убедитесь, что порты `BACK_PORT` и `FRONT_PORT` не заняты другими процессами на вашей хост-машине.

---

## 🚀 2. Быстрый старт

1. Склонируйте репозиторий и перейдите в его корень.
2. Создайте и настройте файл `.env`.
3. Запустите сборку и старт всех контейнеров:
   ```bash
   sudo docker compose up --build
   ```

## 📋 3. Шпаргалка по командам

### Docker Compose (Управление проектом)

Все команды выполняются из корневой папки, где лежит `docker-compose.yml`. Если вы работаете на Linux, перед командами может потребоваться добавить `sudo`.

*   **`docker compose up`** — Запуск проекта в консоли (вы будете видеть логи всех сервисов в реальном времени).
*   **`docker compose up -d`** — Запуск проекта в фоновом режиме (демон).
*   **`docker compose up --build`** — Принудительная пересборка образов (Dockerfile) и запуск. Используйте, если изменились системные зависимости или конфигурация сборки.
*   **`docker compose down`** — Остановка и полное удаление контейнеров и внутренних сетей проекта. Данные в `volumes` при этом сохраняются.
*   **`docker compose down -v`** — Полная очистка: остановка контейнеров + удаление всех связанных `volumes` (очистка кэша, баз данных и т.д.).
*   **`docker compose ps`** — Просмотр статуса запущенных сервисов текущего проекта.
*   **`docker compose logs -f`** — Интерактивный просмотр логов всех контейнеров. Можно указать конкретный сервис, например: `docker compose logs -f testpulseback`.
*   **`docker compose restart`** — Быстрый перезапуск всех сервисов проекта без пересоздания контейнеров.

### Docker (Общие команды для отладки)

*   **`docker ps`** — Список вообще всех запущенных контейнеров на вашем компьютере.
*   **`docker images`** — Список всех скачанных и собранных образов.
*   **`docker exec -it <имя_контейнера> bash`** — Подключиться к терминалу запущенного контейнера (вместо `bash` можно использовать `sh`, если образ на базе Alpine).
    *   *Пример для бэкенда:* `sudo docker exec -it testpulseback_container sh`
*   **`docker stats`** — Мониторинг потребления ресурсов (CPU, RAM, Сеть) контейнерами в реальном времени.
*   **`docker system prune -a --volumes`** — Полная глобальная очистка Docker. Удаляет все неиспользуемые контейнеры, сети, образы и volumes. Помогает, если на диске закончилось место.
